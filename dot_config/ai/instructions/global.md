# Global AI Assistant Instructions

These instructions apply across all AI coding assistants and projects.

## Code Style Preferences

- Use clear, descriptive variable and function names
- Prefer functional programming patterns where appropriate
- Write self-documenting code with minimal comments
- Follow existing code style in the project

## Development Workflow

- Always run tests before committing changes
- Write meaningful commit messages following conventional commits format
- Use feature branches for new work
- Keep commits atomic and focused

## Tool Preferences

- **Package Manager**: Use `mise` for runtime version management
- **Shell**: zsh with antidote plugin manager
- **Editor**: VS Code
- **Git**: Use delta for diffs when available

## Security

- Never commit secrets, API keys, or credentials
- Use environment variables for sensitive data
- Review security implications before running external commands

## Communication Style

- Be concise and direct
- Explain complex decisions
- Ask for clarification when requirements are ambiguous
- Provide context for suggested changes
